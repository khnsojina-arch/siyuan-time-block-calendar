# 时间块日历

一个面向思源笔记的 Notion Calendar 风格插件原型。

## 0.5.6 clean

- 保留左侧番茄钟，只作为计时器使用，不写入专注记录。
- 删除「专注记录」选项卡及相关图表/统计/手动补录逻辑，回到纯时间块日程规划。
- iHour `.ihbak` 解析导出 JSON 移到插件设置里：只下载 JSON，不写入思源文档。
- 导出的 iHour JSON 包含 `records`、`projectSummary`、`dailySummary` 和 `raw` 原始解密数据，方便后续交给 AI 分析。


## 目标

- 周视图：拖拽时间区域创建待办块。
- 月视图：浏览当天有哪些时间块。
- 每个日程对应一个真实思源块。
- 开始/结束时间写入块属性，后续拖动或拉伸时同步更新。
- 默认写入 daily note：`/daily note/YYYY/MM/YYYY-MM-DD`。

## 块属性

插件会给日程块写入这些自定义属性：

```text
custom-calendar-start
custom-calendar-end
custom-calendar-title
custom-calendar-color
custom-calendar-status
```

## 开发

```bash
npm install
npm run build
```

构建产物在 `dist/`。安装到思源时，将 `dist/` 目录复制到思源工作空间的 `data/plugins/siyuan-time-block-calendar`。

## 当前版本

`0.1.0` 是 MVP 骨架：

- 顶栏按钮打开日历标签页。
- 支持周/月视图切换。
- 支持拖拽空白时间创建事件。
- 支持拖动已有事件修改时间。
- 支持点击事件打开对应块。

这个版本会优先验证“日历 UI + 思源块读写”的主链路，后续可以继续做：

- 多日程重叠布局优化。
- 颜色和标签。
- 循环日程。
- 与待办数据库联动。
- AI 自动排程。


## 本次修复

- 左键点击事件块不再弹出右侧属性栏。
- 右键菜单顶部新增「属性」，点击后再打开右侧属性栏。
- 保留右键设置提醒、完成/未完成、复制、删除等操作。

## 0.4.4 电脑系统通知

- 在设置中新增「电脑系统通知」开关。
- 到点提醒时，保留思源内提醒，同时调用桌面端系统通知。
- 新增「授权/测试系统通知」按钮，用于申请通知权限并发送测试提醒。
- 默认避免重新打开思源后大量旧提醒刷屏；需要补发错过提醒的系统通知，可在设置中开启「错过提醒补发」。

注意：系统通知仍依赖思源桌面端运行；思源关闭、电脑关机或睡眠时，插件无法实时检查提醒。

## 0.4.5 完整数据导入导出

- 顶部工具栏新增「导出」按钮，可一键导出完整 JSON 备份。
- 「导入」改为文件选择器导入，不再依赖粘贴文本。
- JSON 备份包含插件设置/标签、全部时间块和提醒信息。
- 导入时默认跳过重复时间块，不会删除现有数据。
- 兼容旧 CSV 文件导入，但 CSV 只包含时间块。


## 0.4.6 当前时间轴修复

- 当前时间红线改为按实际渲染高度计算位置，不再固定使用 56px/小时，避免缩放、样式覆盖后产生偏移。
- 当前时间线每 15 秒自动刷新一次，并在窗口聚焦、切回页面、窗口尺寸变化时立即校准。
- 周/三日视图渲染后延后一帧重新定位时间线，减少刚打开页面时布局尚未稳定导致的位置误差。


## v0.4.7

- 修复周视图/三天视图中时间块拖动后跳回原位置的问题。
- 改用 Pointer Events 与实际渲染高度计算拖拽位置，避免时间换算偏差。
- 时间块新增顶部/底部拖拽手柄，可直接调整开始时间和结束时间。
- 拖拽/缩放后先更新内存状态再刷新当前视图，避免思源索引延迟导致视觉回退。
